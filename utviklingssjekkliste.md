# Utviklingssjekkliste for prosjektet "Utestengt"

Dette dokumentet skal brukes til å følge opp utviklingen av systemet, basert på kravspekk. Huk av hvert punkt etter hvert som det er ferdigstilt.

---

## 1. Foranalyse og arkitektur
- [ ] Teknisk og juridisk kravkartlegging
- [ ] Valg av teknologi og arkitektur
- [ ] Definere datamodell

## 2. Roller og tilgangsstyring
- [ ] Rollebasert tilgangskontroll
- [ ] Autentisering (passord, 2FA)
- [ ] Sikkerhetslogg for sensitive operasjoner

## 3. Kunde- og venue-modul
- [ ] CRUD for kunder
- [ ] CRUD for venues
- [ ] Innstillinger for interne/eksterne nettverk

## 4. Person- og utestengelsesmodul
- [ ] CRUD for personer
- [ ] CRUD for utestengelser
- [ ] Dataminimering og bildeopplasting
- [ ] Automatisk review-dato og statusendringer

## 5. Hendelseslogg
- [ ] Registrering av hendelser
- [ ] Visning for ulike roller

## 6. Søk og visning ved døren
- [ ] Søkefunksjon (navn, fødselsdato/år)
- [ ] Trafikklys-visning
- [ ] Konfigurerbar bildeopplasting/lagring

## 7. Samarbeidsnettverk
- [ ] Modell for interne/eksterne nettverk
- [ ] Deling av utestengelser
- [ ] Logging av oppslag og tilgang

## 8. Personvern og sikkerhet
- [ ] HTTPS og kryptering
- [ ] Passordhashing
- [ ] Sletting/anonymisering med varsling
- [ ] Moduler for innsyn, retting, sletting

## 9. Logging, revisjon og rapportering
- [ ] Revisjonslogg per kunde
- [ ] Eksport av anonymisert statistikk
- [ ] Misbruksdeteksjon og sperring

## 10. Admin-grensesnitt
- [ ] Panel for systemeier
- [ ] Oversikt, drift, statistikk
- [ ] Kunde- og venue-håndtering

## 11. Ansvarsfraskrivelse og vilkår
- [ ] Plattformsavtale og databehandleravtale
- [ ] Rutiner for tvister og misbruk
